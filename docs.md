
## 1. Название
DisctrictPortal (Районный портал)

## 2. Цель
Программа помогает жителям района общаться, обмениваться информацией и решать бытовые вопросы.

## 3. Пользователь
Жители района, председатель ТСЖ/УК, местные предприниматели.

## 4. Функциональные требования (FR)
- FR1: Добавить объявление/новость (название обязательно, категория)
- FR2: Показать ленту объявлений с фильтрами по категориям
- FR3: Добавить комментарий к объявлению
- FR4: Поиск объявлений по ключевым словам

## 5. Нефункциональные требования (NFR)
- Приложение запускается без ошибок
- Возможность загрузки изображений к объявлениям
- Данные хранятся в базе данных (SQLite/PostgreSQL)
- Понятный интерфейс с навигацией по разделам

## 6. Модель данных (черновик)
Объявление Post
- Category (enum: Новость, Проблема, Продажа, Услуга, Потеряшка)
- Content (string, текст объявления)
- Title (string, обязательное)
- Id (int, первичный ключ)
- ImageUrl (string, необязательное)
- Address (string, адрес/дом)
- CreatedAt (DateTime)
- AuthorId (string, ссылка на пользователя)
 Пользователь User
- Id (string)
- Name (string)
- Phone (string, для связи)
- Address (string, дом/квартира)=
Комментарий Comment
- Id (int)
- PostId (int)
- Text (string)
- AuthorId (string)
- CreatedAt (DateTime)
Проблема на карте MapMarker
- Id (int)
- Latitude (double) 
- Longitude (double)
- Description (string)
- PhotoUrl (string)
- Status (enum: Новая, В работе, Решена)


## 7. Критерии приемки (как проверить)
- Можно добавить 3 объявления разных категорий
- Можно найти объявление через поиск
- Можно оставить комментарий к объявлению
- Можно добавить метку проблемы на карту

## 8. План модулей (черновик)
- Core (модели и бизнес-логика)
- Data (работа с БД, Entity Framework Core)
- Web (ASP.NET Core MVC/Razor Pages)
- Services (сервисы: email-уведомления)
- Mapping (интеграция с картами, Yandex Maps API)
